Milestone

Design milestone

A complete blueprint for the learning experience — before a single piece of content is built. The storyboard is the definitive output: it specifies structure, sequence, learning outcomes, assessment approach, and accessibility requirements.

Outputs: assessment report · research report · project plan · storyboard

Activities

  • Content development
  • Technical skill development and aid
  • Beta testing with real learners
  • Structured beta feedback session
  • Quality assurance against accessibility and instructional standards
  • Delivery preparation

Beta feedback is design data, not a formality. QA is documented and accountable. Learner feedback from testing is integrated before delivery.

Milestone

Delivery milestone

Content is ready for deployment. Activities in this phase are grouped to enable reverse-engineering from the integration of learning evaluation data — meaning the evaluation framework is in place before learners begin.

Outputs: beta testing report · quality assurance report · delivery preparation plan

Activities

  • Post-delivery evaluation
  • Learning path evaluation
  • Impact reporting
  • Sustainment planning
  • Update triggers and maintenance cycles
  • Knowledge management

Evaluation is designed from the start — not added at the end. Impact is measured at every level: learner reaction, knowledge acquisition, behavioural transfer, and organisational outcomes.

Milestone

Sustainment milestone

The program continues to work over time. Activities in this phase are grouped from the perspective of the organisation's long-term needs — not just the immediate learning event. Sustainment is planned from the outset, not retrofitted.

Outputs: evaluation and impact report · sustainment plan
